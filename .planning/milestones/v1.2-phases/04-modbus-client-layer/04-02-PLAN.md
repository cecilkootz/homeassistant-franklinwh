pySunSpec2 API (verified from official source — use exactly these):
```python
import sunspec2.modbus.client as modbus_client
from sunspec2.modbus.client import (
    SunSpecModbusClientError,
    SunSpecModbusClientTimeout,
    SunSpecModbusClientException,
)

# Device construction
device = modbus_client.SunSpecModbusClientDeviceTCP(
    slave_id=slave_id,
    ipaddr=host,
    ipport=port,
)

# Connection + scan pattern (MUST use connect=False in scan since we called connect() separately)
device.connect()
device.scan(connect=False, full_model_read=False)

# Model read pattern
device.models[713][0].read()

# Value extraction — ALWAYS use .cvalue (scale-factor-applied float), never .value
battery_soc = float(device.models[713][0].SoC.cvalue)        # Model 713: SoC point, Pct
grid_ac_power = float(device.models[701][0].W.cvalue)         # Model 701: W point, Watts
battery_dc_power = float(device.models[714][0].DCW.cvalue)    # Model 714: DCW point, Watts (may be None — see fallback below)
solar_power = float(device.models[502][0].OutPw.cvalue)       # Model 502: OutPw point, Watts

# Model 714 DCW fallback (if model-level DCW is None, sum the repeating port group):
dcw_val = device.models[714][0].DCW.cvalue
if dcw_val is None:
    dcw_val = sum(
        float(port.DCW.cvalue)
        for port in device.models[714][0].groups.get("DCSrc", [])
        if port.DCW.cvalue is not None
    )

# disconnect always in finally block
device.disconnect()
```

Sign conventions (SunSpec native — do NOT negate any values):
- Model 701 W: positive = DER generation / export to grid; negative = import from grid
- Model 714 DCW: positive = discharge; negative = charge
- Model 502 OutPw: positive = solar production
- home_load formula: solar_power + battery_dc_power - grid_ac_power
